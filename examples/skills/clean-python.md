---
description: Enforce clean Python coding style — no unnecessary comments, imports at top, no docstrings unless complex
---

When writing Python code:

- Never add unnecessary comments or docstrings. Code should be self-explanatory.
- Never add comments like "# This function does X" before obvious code.
- Never add docstrings just because a function exists.
- Always put imports at the top of the file, grouped: stdlib, third-party, local.
- Use type hints on function signatures.
- No wildcard imports.
